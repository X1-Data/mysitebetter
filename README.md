# http method
   GET: ใช้สำหรับ ขอหรือดึงข้อมูล จาก Server ข้อมูลที่ได้จะแสดงต่อท้าย URL ไม่มีผลในการเปลี่ยนแปลงข้อมูลบน Server
   POST: ใช้สำหรับ ส่งข้อมูล ไปยัง Server มักใช้เมื่อกรอกแบบฟอร์ม (Form) หรืออัปโหลดไฟล์
   PUT: ใช้สำหรับ อัปเดตหรือสร้างข้อมูลใหม่ บน Server หากข้อมูลเดิมมีอยู่แล้วจะเป็นการแทนที่ (Replace) ข้อมูลนั้นทั้งหมด
   PATCH: ใช้สำหรับ แก้ไขข้อมูลบางส่วน บน Server (คล้าย PUT แต่อัปเดตเฉพาะฟิลด์ที่ส่งไป ไม่แทนที่ของเดิมทั้งหมด)
   DELETE: ใช้สำหรับ ลบข้อมูล ออกจาก Server
   Method เฉพาะทาง
   HEAD: คอนเซปต์เดียวกับ GET แต่เซิร์ฟเวอร์จะคืนค่ากลับมาเฉพาะ Header โดยไม่มี 
   BodyOPTIONS: ใช้ตรวจสอบว่า Server หรือ API นั้นรองรับ Method ใดบ้าง 
   TRACE: ใช้สำหรับทดสอบการสื่อสารปลายทาง โดย Server จะสะท้อน Request กลับมาให้ Client เพื่อตรวจสอบเส้นทาง
